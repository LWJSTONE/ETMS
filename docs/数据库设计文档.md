## 表结构设计

### 1. 用户表 (sys_user)

存储系统用户信息，包括管理员和普通员工。与实体类 `com.etms.entity.User` 对应。

| 字段名 | 数据类型 | 长度 | 允许空 | 默认值 | 说明 |
|--------|---------|------|-------|--------|------|
| id | BIGINT | 20 | 否 | 自增 | 用户ID（主键） |
| username | VARCHAR | 50 | 否 | - | 登录用户名 |
| password | VARCHAR | 100 | 否 | - | 密码（BCrypt加密） |
| real_name | VARCHAR | 50 | 否 | - | 真实姓名 |
| gender | TINYINT | 1 | 是 | 0 | 性别(0未知/1男/2女) |
| email | VARCHAR | 100 | 是 | - | 电子邮箱 |
| phone | VARCHAR | 20 | 是 | - | 手机号码 |
| avatar | VARCHAR | 255 | 是 | - | 头像URL |
| dept_id | BIGINT | 20 | 是 | - | 部门ID |
| position_id | BIGINT | 20 | 是 | - | 岗位ID |
| status | TINYINT | 1 | 是 | 1 | 状态(0禁用/1正常/2离职/3休假) |
| login_ip | VARCHAR | 50 | 是 | - | 最后登录IP |
| login_time | DATETIME | - | 是 | - | 最后登录时间 |
| pwd_expire_time | DATETIME | - | 是 | - | 密码过期时间 |
| lock_time | DATETIME | - | 是 | - | 账号锁定时间 |
| lock_count | INT | 11 | 是 | 0 | 连续登录失败次数 |
| create_by | BIGINT | 20 | 是 | - | 创建人ID |
| create_time | DATETIME | - | 否 | CURRENT_TIMESTAMP | 创建时间 |
| update_by | BIGINT | 20 | 是 | - | 更新人ID |
| update_time | DATETIME | - | 是 | - | 更新时间 |
| remark | VARCHAR | 500 | 是 | - | 备注 |

**索引设计：**
- 主键索引：id
- 唯一索引：username, email, phone
- 普通索引：dept_id, status

## 数据库脚本说明

### 脚本文件位置

- 建表脚本：`etms-backend/src/main/resources/schema.sql`
- 初始数据：`etms-backend/src/main/resources/data.sql`

### 执行方式

```bash
# Windows环境下执行（使用强制编码参数避免乱码）
mysql -u root -p123456 --default-character-set=utf8mb4 < schema.sql
mysql -u root -p123456 --default-character-set=utf8mb4 < data.sql
```

### 注意事项

1. 脚本开头已添加 `DROP DATABASE IF EXISTS etms;` 语句，执行时会删除已存在的同名数据库
2. 默认数据库root密码为 `123456`
3. 所有用户初始密码为 `123456`（BCrypt加密存储）
